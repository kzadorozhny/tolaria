## Context

Tolaria today is a Tauri v2 desktop app per [ADR-0001](0001-tauri-react-stack.md): a Rust shell (`src-tauri/`, ~134 `#[tauri::command]` functions across ~10 modules) hosts a React 19 + Vite frontend (~283 components in `src/`). Every chrome surface — sidebar, note list, inspector, command palette, status bar, settings, ~17 dialogs, ~20 popovers, banners — runs inside a single WKWebView, side by side with the BlockNote + CodeMirror editor body ([ADR-0022](0022-blocknote-rich-text-editor.md)).

This architecture carries growing costs:

- The whole UI shares one WebView, paying its memory, startup, and accessibility tax for surfaces that have no reason to be HTML/CSS.
- Native macOS feel (menus, traffic lights, IME, keyboard, focus) lives only through bespoke workarounds layered over Tauri: [ADR-0052](0052-renderer-first-shortcut-execution-with-native-menu-dedupe.md) (renderer-first shortcut execution with native-menu dedupe), [ADR-0053](0053-webview-init-prevention-for-browser-reserved-shortcuts.md) (webview-init prevention for browser-reserved shortcuts), [ADR-0104](0104-tauri-frontend-readiness-watchdog.md) (Tauri frontend readiness watchdog), [ADR-0079](0079-linux-window-chrome-and-menu-reuse.md) (Linux window-chrome reuse).
- Cross-platform releases ([ADR-0080](0080-cross-platform-desktop-release-artifacts-and-portable-vault-names.md), [ADR-0083](0083-dual-architecture-macos-release-artifacts.md)) are constrained by Tauri's bundler.
- A second observation: the editor body is the only part of the app that genuinely needs a WebView. Everything else is chrome that a native UI runtime renders better.

Zed's `gpui` framework ([zed-industries/zed](https://github.com/zed-industries/zed)) is a production-grade native UI runtime in Rust; `gpui-component` ([longbridge/gpui-component](https://github.com/longbridge/gpui-component), v0.5.2, Apache-2.0) provides shadcn-style primitives on top of it. Together they make a native chrome with a preserved JS editor body tractable.

## Decision

**Replace Tauri with a native GPUI shell. Keep BlockNote + CodeMirror as a per-note `WKWebView` embedded inside a GPUI `Item`. Adopt `gpui-component` as the primitive layer. Ship macOS only at v1.**

Specifically:

1. **Cargo workspace** mirroring Zed's `crates/` layout. Top-level crates: `tolaria` (binary), `tolaria_workspace` (root container), `tolaria_actions` / `tolaria_settings` / `tolaria_theme` / `tolaria_ui` (chrome scaffolding), `vault` / `vault_watcher` / `vault_search` / `git_provider` / `cli_agents` / `mcp_bridge` / `telemetry` / `app_updater` / `localization` (services), `sidebar_panel` / `note_list_pane` / `inspector_panel` (panels), `note_item` / `editor_bridge` / `editor_host_assets` (editor host), `command_palette` / `quick_open` / `dialogs` / `banners` / `startup` / `diff_view` (modal/transient surfaces). `src-tauri/` is removed.

2. **Services replace IPC.** The ~134 `#[tauri::command]` functions become methods on a small set of `Global` services and `Entity<T>` handles: `VaultStore`, `GitService`, `SearchService`, `AiAgentRuntime`, `McpBridge`, `SettingsStore`, `VaultRegistry`, `Telemetry`, `AppUpdater`, `LocalizationStore`, `Clipboard`, `FileDialogService`. Views call them directly; long-running ops return `Task<Result<T>>`; updates flow through `EventEmitter` + `cx.observe`. Existing Rust modules in `src-tauri/src/{vault,git,frontmatter,search,vault_watcher,vault_list,settings,mcp,app_updater}.rs` and the CLI agent runtimes (`cli_agent_runtime.rs`, `claude_cli.rs`, `codex_cli.rs`, `gemini_cli.rs`, `pi_cli.rs`, `opencode_cli.rs`) are repackaged into their target crates with internals mostly intact — only the Tauri command shims and event-emit calls change.

3. **Workspace topology** modeled on `zed::Workspace`. `TolariaWorkspace` owns three Docks (Left = navigation: types + saved views + folders; Right = inspector: properties + outline + AI chat; Bottom = search results + git history, collapsed by default), a center `PaneGroup` with `NoteListPane | EditorPane`, plus `StatusBar`, `ModalLayer`, `ToastLayer`, and the native macOS title bar. Each open note is an `Item` (`NoteItem`) whose render lays out a breadcrumb + persistent banners + an `EditorBodyView` that owns a child `WKWebView`. [ADR-0003](0003-single-note-model.md) (single note open at a time) is revisited: GPUI's `Pane` provides multi-tab for free, and we adopt it.

4. **Editor embedding.** `EditorBodyView` hosts a `WKWebView` created via the `wry` crate (`WebViewBuilder::new_as_child(&parent_view)`) as a sibling `NSView` of GPUI's Metal renderer inside the same `NSWindow.contentView` (confirmed safe at [`crates/gpui_macos/src/window.rs:783–884`](https://github.com/zed-industries/zed/blob/147524879e328513b0883017733462b03b27cf02/crates/gpui_macos/src/window.rs#L783-L884)). Frame sync piggy-backs on layout events (`PaneResized`, window resize, sidebar toggle) with an epsilon compare. One `WKWebView` per open note, LRU-capped at 10; tab switch is `set_hidden` + z-order raise, not navigation. Static assets ship via wry's `with_custom_protocol("tolaria-editor", ...)` reading from `rust-embed`'d `editor-host/dist/`; dev mode proxies to Vite at `http://localhost:5173`.

5. **`editor-host/`** is a separate Vite project containing ~15-25 carried-over files: `Editor.tsx` trimmed to single-doc, `EditorContent.tsx`, `editorSchema.tsx`, `RawEditorView.tsx`, `editorRawModeSync.ts`, `tolariaEditorFormatting.tsx`, `editorLinkAutolink.ts`, the math/Mermaid/tldraw/code-block extension files (preserving [ADR-0022](0022-blocknote-rich-text-editor.md), [ADR-0063](0063-blocknote-code-block-package-for-editor-highlighting.md), [ADR-0082](0082-markdown-durable-math-notes.md), [ADR-0088](0088-markdown-durable-mermaid-diagrams.md), [ADR-0107](0107-markdown-durable-tldraw-whiteboards.md)), `useSaveNote.ts`, `useEditorContentModel.ts`, `useImageDrop.ts`, `useEditorFocus.ts`, plus a new `bridge.ts`. Every `invoke('cmd', ...)` becomes `window.tolariaShell.*` over a JSON envelope `{ v: 1, id?, kind: 'event'|'request'|'response', type, payload?, error? }` routed via wry's IPC handler (JS→Rust) and `WebView::evaluate_script` (Rust→JS). Tldraw, Mermaid, math, and image lightbox stay inside the WebView as part of the document body.

6. **Native macOS menu dispatches all chrome shortcuts.** `cx.set_menus(app_menus(cx))` installs an `NSMenu` whose key equivalents (Cmd+S, Cmd+K, Cmd+P, Cmd+N, …) macOS routes before firstResponder, so they fire even while the WKWebView holds focus. Editor-internal chords (Cmd+B, Cmd+/, Tab) remain JS-handled. The renderer-first / webview-prevent dance from [ADR-0052](0052-renderer-first-shortcut-execution-with-native-menu-dedupe.md) / [ADR-0053](0053-webview-init-prevention-for-browser-reserved-shortcuts.md) is no longer necessary: there is one focused surface at a time and the menu wins outright.

7. **`gpui-component` v0.5.2** supplies primitives: Button, Input, Switch, Select, Slider, Dialog, Sheet, Popover, Tooltip, Notification, Alert, Tab, Sidebar, Resizable, ScrollArea, List, Table, Tree, Avatar, Badge, Tag, Calendar, DatePicker, ColorPicker, Accordion, Breadcrumb, Skeleton, Spinner, Progress, TitleBar. Tolaria-specific compounds (EmojiPicker, IconPicker over Phosphor, CommandPalette delegate, QuickOpen delegate, InspectorRow, FilterPill, WikilinkCombobox, RichTooltip-with-shortcut) live in `crates/tolaria_ui`. Top-level structure (`Workspace`, `Panel`, `Item`, `ModalView`, `Picker`) comes from Zed crates. `gpui-component` is pinned to a specific version.

8. **Platform scope.** v1 ships macOS only. Windows and Linux are deferred behind feature flags; [ADR-0079](0079-linux-window-chrome-and-menu-reuse.md), [ADR-0080](0080-cross-platform-desktop-release-artifacts-and-portable-vault-names.md), [ADR-0083](0083-dual-architecture-macos-release-artifacts.md) are temporarily descoped pending native-shell stabilization. [ADR-0005](0005-tauri-ios-for-ipad.md) (Tauri iOS for iPad) becomes inapplicable until a separate iPad strategy is decided.

9. **Migration shape.** Big-bang rewrite inside the same repo. The new `crates/` workspace and `editor-host/` are built up phase by phase under the same branch policy ([ADR-0021](0021-push-to-main-workflow.md): push to main; no PRs); intermediate builds are dogfood-only. The single cut-over release deletes `src-tauri/` and prunes `src/` to the editor-host carry-overs. A Phase 0 spike crate (`crates/embed_poc/`) validates `WKWebView`-in-GPUI focus, IME, frame sync, and Cmd+S delivery before further crates are written; spike failure forces re-evaluation of this decision (see Consequences).

## Alternatives considered

- **Native GPUI shell with embedded JS editor (chosen)**: minimum churn on the editor body; Tolaria-specific BlockNote schema (wikilinks, slash menu, math, Mermaid, tldraw, code blocks) keeps working as-is; cost is the WKWebView embedding bridge plus IME/focus arbitration. Recovers native macOS UI quality without rewriting the editor.
- **Native GPUI shell with native GPUI editor**: rewrite the editor on top of Zed's `crates/editor/` + custom wikilink/math/Mermaid/tldraw primitives. Architecturally cleaner — Zed embeds zero WebViews — and eliminates the embedding bridge. Rejected for v1 because reimplementing BlockNote-level behavior is months of work and a regression risk on the most-touched user surface. Worth revisiting once chrome stabilizes.
- **Stay on Tauri, lean harder into native chrome via Tauri menu/window plugins**: lowest disruption; preserves [ADR-0001](0001-tauri-react-stack.md) / [ADR-0052](0052-renderer-first-shortcut-execution-with-native-menu-dedupe.md) / [ADR-0053](0053-webview-init-prevention-for-browser-reserved-shortcuts.md) / [ADR-0079](0079-linux-window-chrome-and-menu-reuse.md) / [ADR-0104](0104-tauri-frontend-readiness-watchdog.md) as-is. Does not lift the qualitative UI ceiling that motivates the move and leaves the WebView-for-everything cost in place.
- **Electron + native menus**: same WebView-for-everything model; offers no improvement over Tauri.
- **SwiftUI rewrite (macOS-only)**: maximum native feel, but locks Tolaria to Apple platforms, discards the Rust backend, and forces a full editor rewrite anyway. Rejected for the same reasons as in [ADR-0001](0001-tauri-react-stack.md).
- **Incremental dual-build (some surfaces GPUI, some Tauri/React, gradual swap)**: avoids one big switchover but doubles infrastructure (two windowing models, two state systems, two test rigs) for months. Rejected as the most complex path.
- **Separate `tolaria-rs` repo**: cleanest isolation; rejected because it doubles maintenance until parity and fragments the ADR history.

## Consequences

- The codebase shape inverts from "Rust shell hosts a React app" to "Native Rust app hosts a small React editor". A new Cargo workspace replaces `src-tauri/`; most of `src/` is deleted; `editor-host/` carries ~20 surviving editor-body files.
- Several active ADRs are superseded or descoped by this move and need follow-up status flips: [ADR-0001](0001-tauri-react-stack.md) (Tauri+React stack — superseded), [ADR-0003](0003-single-note-model.md) (single-note model — revisited; multi-tab adopted), [ADR-0005](0005-tauri-ios-for-ipad.md) (Tauri iOS — inapplicable at v1), [ADR-0030](0030-rust-commands-module-split.md) (Rust commands module split — superseded by entity services), [ADR-0052](0052-renderer-first-shortcut-execution-with-native-menu-dedupe.md) / [ADR-0053](0053-webview-init-prevention-for-browser-reserved-shortcuts.md) (shortcut routing — superseded; native menu dispatches first), [ADR-0079](0079-linux-window-chrome-and-menu-reuse.md) / [ADR-0080](0080-cross-platform-desktop-release-artifacts-and-portable-vault-names.md) / [ADR-0083](0083-dual-architecture-macos-release-artifacts.md) (Linux/cross-platform artifacts — descoped behind feature flags), [ADR-0104](0104-tauri-frontend-readiness-watchdog.md) (Tauri frontend readiness watchdog — no longer applicable), [ADR-0106](0106-shared-app-command-manifest.md) (shared app command manifest — replaced by `actions!()` + keymap JSON).
- ADRs that describe filesystem/vault/Git/AI behavior continue unchanged because they are independent of the UI runtime: [ADR-0002](0002-filesystem-source-of-truth.md), [ADR-0014](0014-git-based-vault-cache.md), [ADR-0022](0022-blocknote-rich-text-editor.md) (editor body survives), [ADR-0040](0040-custom-views-yml-filter-engine.md), [ADR-0067](0067-autogit-idle-and-inactive-checkpoints.md), [ADR-0077](0077-concurrent-safe-vault-cache-replacement.md), [ADR-0089](0089-active-vault-filesystem-watcher.md), [ADR-0093](0093-shared-cli-agent-runtime-adapters.md), [ADR-0099](0099-cumulative-vault-asset-scope.md), [ADR-0111](0111-path-aware-external-vault-refresh-with-focused-editor-preservation.md), [ADR-0114](0114-mounted-workspaces-unified-graph.md), and the locale/telemetry ADRs 0016, 0084, 0087, 0101.
- Performance and feel improve: native menus, native focus and IME, native scroll, native fonts, GPU-accelerated chrome render, one WKWebView per open note instead of a monolithic WebView for everything.
- Memory profile shifts: ~50 MB resident per open-note WKWebView (capped at 10 by LRU) vs. today's single-WebView baseline. Net working set is expected to drop because chrome no longer pays a WebView tax; this is a measurement gate before cut-over.
- IPC complexity is bounded to the `editor_bridge` envelope between Rust and the editor-host React app. Every other code path runs as a direct Rust call against a service entity.
- Dependency surface gains `gpui` (git-pinned), `gpui-component` (commit-pinned, pre-1.0), `wry`, `rust-embed`, `rust-i18n`, `sentry`. The `gpui-component` API churn risk is mitigated by pinning and is the chief stability gamble.
- Release pipeline changes: Tauri bundler is replaced by a Zed-style `script/bundle-mac` driving codesign and notarization. Apple Developer cert plumbing must be re-wired.
- Test strategy splits in two: editor-body Playwright specs in `tests/smoke/` keep running against the embedded WKWebView (they are about BlockNote/CodeMirror behavior); chrome smoke specs are rewritten as native GPUI tests + macOS QA scripts. The ~129 existing specs are partitioned at cut-over.
- The CodeScene gate per [ADR-0064](0064-ratcheted-codescene-thresholds.md) applies to the new Rust crates; `.codescene-thresholds` is reset against the first push of the new code so subsequent ratcheting continues from a real baseline.
- Re-evaluation triggers:
  - **Phase 0 spike failure.** If WKWebView focus handoff, IME mid-composition, or frame sync during sidebar drag cannot be made stable on macOS, the embedded-JS option becomes untenable and the native-GPUI-editor alternative is reopened.
  - **Cross-platform complexity** and the deferred Windows/Linux work blocks adoption beyond what a feature-flagged shim can absorb.
  - **iPad / iOS strategy** is reactivated (per [ADR-0005](0005-tauri-ios-for-ipad.md)); GPUI on iOS is not a current target and a different stack would need to be reconsidered.

## Advice

Three exploration agents mapped (i) Tolaria's full chrome inventory and editor↔shell boundary across `src/` and `src-tauri/`, (ii) Zed's GPUI patterns for Workspace, Dock, Panel, Item, ModalView, Picker, ContextMenu, StatusBar, Settings, and `set_menus`, and (iii) the embedding option space across raw `WKWebView` via `objc2`, the `wry` crate, pure-native GPUI editor, and a hybrid. Three plan agents then converged on the same shape: workspace topology, ~85 service methods replacing 134 commands, and the wry-as-child-NSView embedding path. A fourth exploration evaluated [longbridge/gpui-component](https://github.com/longbridge/gpui-component) against Tolaria's primitive needs and rated it as covering 16/26 directly, 6 partially, 4 missing — leading to selective adoption. User direction set the hard constraints: macOS only at v1, big-bang same-repo rewrite, keep the JS editor, keep tldraw/Mermaid/math/lightbox inside the WebView.
